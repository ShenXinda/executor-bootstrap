## Purpose

统一管理16个共享槽位的坐标点，支持告警点和清扫点的添加、取消和自动回收。

## 新增需求

### 需求：添加告警坐标点

当槽位数量少于16个且坐标点不存在时，系统必须允许向ZoneManager添加告警坐标点。

#### 场景：成功添加告警点

- **当** 调用AddPoint传入ZoneType::Alert和坐标(x, y)，且该坐标不存在且总点数少于16个
- **则** 添加成功并返回true

#### 场景：槽位已满时添加告警点

- **当** 已有16个点存在时调用AddPoint传入ZoneType::Alert
- **则** 不添加并返回false

#### 场景：添加重复告警点

- **当** 调用AddPoint传入ZoneType::Alert和坐标(x, y)，但该坐标已存在
- **则** 不添加并返回false

### 需求：添加清扫坐标点

当槽位数量少于16个且坐标点不存在时，系统必须允许向ZoneManager添加清扫坐标点。

#### 场景：成功添加清扫点

- **当** 调用AddPoint传入ZoneType::Clean和坐标(x, y)，且该坐标不存在且总点数少于16个
- **则** 添加成功并返回true

### 需求：移除坐标点

系统必须允许通过坐标移除已存在的坐标点。

#### 场景：成功移除坐标点

- **当** 调用RemovePoint传入已存在的坐标(x, y)
- **则** 移除成功并返回true

#### 场景：移除不存在的坐标点

- **当** 调用RemovePoint传入不存在的坐标(x, y)
- **则** 返回false

### 需求：清扫时自动回收

当机器人位置匹配清扫点时，系统必须自动释放该槽位资源。

#### 场景：清扫点触发自动回收

- **当** 调用OnPositionChanged传入与清扫点匹配的位置
- **则** 移除该清扫点并回收槽位

### 需求：查询区域状态

系统必须提供查询当前区域状态的方法，包括空闲槽位数、告警点数和清扫点数。

#### 场景：查询空闲槽位数

- **当** 调用GetFreeCount
- **则** 返回可用槽位数（16 - alertCount - cleanCount）

#### 场景：查询告警点数

- **当** 调用GetAlertPointCount
- **则** 返回已配置的告警点数

#### 场景：查询清扫点数

- **当** 调用GetCleanPointCount
- **则** 返回已配置的清扫点数
